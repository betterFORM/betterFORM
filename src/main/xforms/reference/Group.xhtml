The group Element
Description

This form shows how betterForm interprets the XForms 'appearance' attribute for groups. CSS rules to influence the standard layout as shown below can be influenced by modifying or overwriting style rules in betterform.css. Please resize your browser window to see the behavior of certain appearances.

Description

A group without any 'appearance' attribute defaults to a rendering equivalent to a compact group (see below). Though no appearance is specified this behavior is a concession to the lazy or unexperienced author thereby getting a clean readable output.

Minimal Group

A group with 'minimal' appearance simply wraps the contents of the group in a span Element in the rendered output and is layout-agnostic. It should be used when no layout is implied by the group but only a XForms binding scope should be established.

XForms Markup

<!--XForms UI-->
<xf:group ref="grouped-data/group" appearance="minimal">
    <xf:label>Group with minimal appearance (layout-agnostic)</xf:label>
    <xf:input ref="a">
        <xf:label>A-label</xf:label>
    </xf:input>
    <xf:input ref="b">
        <xf:label>Bb-label</xf:label>
    </xf:input>
    <xf:input ref="c">
        <xf:label>Ccc-label</xf:label>
    </xf:input>
    <xf:input ref="d">
        <xf:label>Dddd-label</xf:label>
    </xf:input>
</xf:group>

Compact Group

A compact group renders the labels of any containing control above the actual control and therefore provides a very narrow layout of a group.

    <xf:label>
        <b>Compact Group</b>
        uses a horizontal table layout
    </xf:label>
    <xf:input ref="a">
        <xf:label>A-label</xf:label>
    </xf:input>
    <xf:input ref="b">
        <xf:label>B-label</xf:label>
    </xf:input>

        <xf:label>C-label</xf:label>
    </xf:input>
    <xf:input ref="d">
        <xf:label>D-label</xf:label>
    </xf:input>
</xf:group>

Vertical Group (rendered as HTML Table)

A group with the custom appearance 'bf:verticalTable' is largely the same as a full group except that is uses tables for left/right aligning labels and controls. This in some cases makes it easier to nest groups as well as optimize the space the respective labels take. With CSS you need to specify a fixed width for the labels which is problematic when labels differ a lot in length. Further labels that do not fit the specified width may corrupt your layout. The use of tables avoids those problems.

XForms Markup

<!--XForms UI-->
<xf:group ref="grouped-data/group" appearance="bf:verticalTable">
    <xf:label>
        <b>Vertical Group</b>
        rendered as HTML Table
    </xf:label>
    <xf:input ref="a">
        <xf:label>A-label</xf:label>
    </xf:input>
    <xf:input ref="b">
        <xf:label>Bb-label</xf:label>
    </xf:input>
    <xf:input ref="c">
        <xf:label>Ccc-label</xf:label>
    </xf:input>
    <xf:input ref="d">
        <xf:label>Dddd-label</xf:label>
    </xf:input>
</xf:group>
