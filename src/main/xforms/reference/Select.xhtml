The select Control
Description

This form control allows the user to make multiple selections from a set of choices. All select controls on this page are bound to the same instance. Thus if you change the value of one control the others will update acccordngly.

XForms 1.1 Links

Quick Reference
Select Control with appearance 'minimal'

If no appearance is given, the XForms select control defaults to a HTML select control.

XForms Markup

<!--XForms UI-->
<xf:select id="select1" ref="item/value">
    <xf:label>Country</xf:label>
    <xf:itemset nodeset="instance('codes')/country">
        <xf:label ref="country-name"></xf:label>
        <xf:value ref="code"></xf:value>
    </xf:itemset>
</xf:select>

Standart Select Control Appearance 'full'

Select with appearance="full" will be rendered as a set of checkboxes.

XForms Markup

<!--XForms UI-->
<xf:select id="select2" ref="item/value" appearance="full">
    <xf:label>Country</xf:label>
    
    <xf:hint>This is a standard select control</xf:hint>
    <xf:help>Not much to say here but some basic html formatting
        is

Standard Select Control

A select with attribute 'selection="open"' allows the user to add an entry that is not contained in the predefined list of choices. It will be rendered as a select along with an input control. The selection attribute takes precedence over the appearance attribute.

XForms Markup

<!--XForms UI-->
<xf:select id="select3" ref="item/value" selection="open">
    <xf:label>Country</xf:label>
    
    <xf:hint>This is a standard select control</xf:hint>
    <xf:help>Not much to say here but some basic html formatting
        is
        <b>possible</b>
    </xf:help>
    <xf:alert>The select value is not valid</xf:alert>
    <xf:itemset nodeset="instance('codes')/country">
        <xf:label ref="country-name"></xf:label>
        <xf:value ref="code"></xf:value>
    </xf:itemset>
</xf:select>
